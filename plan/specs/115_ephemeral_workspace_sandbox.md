# SPEC-115: Ephemeral Workspace Isolation Layer & Sandbox Cloner

## Summary
Create a comprehensive, copy-on-write virtual workspace sandbox cloner (`scripts/agent/workspace-clone.js` and `src/net/EphemeralSandbox.js`) to provision highly-isolated, dynamic, and transient directories for untrusted AI agent execution runs.

## Motivation
- Unattended, multi-day engineering runs suffer from workspace dirtiness, where files generated by previous runs interfere with downstream tasks, resulting in test contamination and state leakage.
- Enforces the P0 Security pillar by preventing untrusted guest executions from accessing or mutating root host configuration files, environment variables, or private local directories.

## Scope
**In:**
- Build `scripts/agent/workspace-clone.js` establishing an ephemeral copy-on-write directory (e.g., `.sandbox-worktrees/run-XYZ`) containing ONLY the clean git-tracked repository baseline (no uncommitted configurations, secrets, or temporary logs).
- Integrate `src/net/EphemeralSandbox.js` to dynamically mount, monitor, and clean up these directories.
- Monkey-patch path resolvers inside `ProcessSentinel` to restrict directory mutation strictly to the temporary ephemeral workspace directory, throwing isolation boundary faults on any escape attempts.
- Ensure the cloner automatically purges the workspace on task completion.

**Out:**
- Do not utilize third-party OS virtualizers (like Docker or VirtualBox) to preserve zero external runtime dependencies.

## Acceptance Criteria
- [ ] Cloner successfully provisions an ephemeral directory from git-tracked files.
- [ ] ProcessSentinel throws a security boundary fault when a task attempts to write outside the cloned path.
- [ ] ephemerality is verified by confirming the cloned path is completely deleted post-execution.
- [ ] `npm run agent:check` green.
